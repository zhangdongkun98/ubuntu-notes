最近我在学习的工程中将我的 Ubuntu 18.04 操作系统做了一次彻底的优化和美化。为什么要美化和优化呢？美化是为了让自己吃饭的工具看起来漂亮，毕竟每天都要使用的；优化当然是让系统的运行工作得更好。现在我在我在虚拟机中重新安装了 Ubuntu 18.04 desktop 操作系统，准备把整个美化的过程记录下来。

下文中的资料我已经放百度云了，真心要的朋友私信我获取提取码。

1. 安装操作系统

  • 物理机安装参考：
https://zhuanlan.zhihu.com/p/58410978​zhuanlan.zhihu.com
  • 这里需要说明一点，在这篇文章的介绍中，我建议将硬盘分多个分区，便于在系统出现故障的时候恢复系统，但是现在我找到了另外的恢复系统的方法，而且更好用，所以这里我就建议不用分那么多个分区了，直接分出 100G - 200G 大小的空间挂在在 / 目录下，用于安装系统，剩下的全部分到一个区，比如我的剩下磁盘就分一个区，挂在在 /file 目录下，用于存储资料。

安装完系统，重启计算机，如下图：

登录进系统看看新安装的 Ubuntu 18.04 桌面是怎么样的吧，后面优化了之后再来对比一下，看看变化，真的是必须美化呀，不然默认的桌面会 XX 我的脑部神经的。

上图我打开了终端，使用 df -h 命令查看了我硬盘的分区和使用情况，可见，我一共两块硬盘，第一块 /dev/sda 挂载在 / 分区下，第二块 /dev/sdb 挂载在 /file 下。下面开始正式开始美化的步骤。

2. Ubuntu 18.04 桌面主题美化

2.1. 更换最优软件更新源

Ubuntu 有自己的软件仓库，这些软件仓库分布在全世界各地，其中的软件都是一样的，当然国内的软件仓库源可能会有延后性，但这个延后的时间不会太久，一般不超过一天。为什么要更换软件源地址呢？因为 Ubuntu 18.04 默认使用的是国外的软件源地址，在国内如果不更换软件源地址的话会导致更新软件奇慢无比。那么怎么更换软件源呢？我之前在学习的时候，看网上很多教程都是把国内的软件源的地址找到，然后编辑 /etc/apt/sources.list 这个文件来更换软件源，但这样我哪知道找到的那个软件源地址是不是最优的呢？现在不用纠结，有了更好的办法，只需要打开 software & updates 这个软件，点点鼠标就可以了，具体如下图：

打开 software & updates 软件后如上图，当点击 select best server 后，等待一会儿系统就会自动选择最优的软件源，点击 choose server ，系统提示输入密码，输入密码确认就可以了，如下图：

2.2. 安装显卡驱动

显卡驱动最简单的安装方法就是在 software & updates 软件中，Additional Drivers 选项下，系统会自动检测你电脑的显卡驱动，点击选择你显卡的驱动应用就可以了，如下图：

上图中显示没有显卡驱动可用，因为我是在 虚拟机中安装的系统，所以显示没有，下面我贴一张我物理机上的图：

如果你还是不清楚怎么安装显卡驱动，请先确认你的显卡是不是 NVIDIA 的显卡，如果是，可以参考我这篇文章安装：Ubuntu 18.04 安装 NVIDIA 显卡驱动 ，如果你是其他显卡，通过上面的步骤还是不能安装的，请自行网上找教程，或者与我联系。不要想着不安装了，显卡驱动是必须要安装的，这个会影响后面的美化的显示效果。

2.3. 登录 Ubuntu 的云端账号

Ubuntu 在发布版本后会不定期的推送一些系统内核级别的补丁，如果我们登录了 Ubuntu 的云端账号，并且启用了自动安装这些补丁的功能的话，只要 Ubuntu 官方发布了新的补丁，我们的系统就会自动安装，设置的界面如下，需要自行先注册 Ubuntu 的云端账号。注册地址是：https://login.ubuntu.com/+login

2.10. 更改 gnome shell 主题

  • macOS MOJAVE： https://www.gnome-look.org/p/1213208/，这个是我选择的 gnome shell 主题，大家也可以在 Gnome Shell Themes 类目下选择自己喜欢的 gnome shell 主题。这个 gnome shell 主题的安装方法和上面其他主题的安装方法有点不一样，还比较复杂，一般在相应的主题页面或者解压后的文件夹里面都会有安装说明，如果你选择的也是这个 gnome shell 主题，可以按照我的方法安装，如果是其他主题，那你需要认真阅读相应主题提供的安装方法进行安装。
  • 下载上图中圈出的两个文件
  • 解压 OSX.for.Dash.to.PANEL .tar，进入解压后的目录 OSX.for.Dash.to.PANEL .tar/OSX.for.Dash.to.PANEL/FONT，这个文件夹下有两个字体文件，文件名都是 San 开头的，将其复制到 ~/.local/share/fonts/ 目录下，没有这个目录就自己创建一下。
  • 打开 settings 软件，在 Details 选项下，看看如下图的 gnome 的版本，Ubuntu 18.04 目前使用的是 3.28.2 版本。如果你的大版本不是 3.28 ，那接下来的操作就要按照你的版本相应变化。
  • 复制 OSX.for.Dash.to.PANEL .tar/OSX.for.Dash.to.PANEL/3.26 - 3.28/ 这个目录下的 Mojave 整个文件夹到 .themes 目录下
  • 复制 ~/.themes/Mojave-dark/gnome-shell/assets/activities.svg 到 ~/.themes/Mojave/gnome-shell/assets 目录下
  • 打开 ~/.themes/Mojave/gnome-shell/gnome-shell.css 文件，在下图所示的位置加入如下内容，下图是已经加入后的截图
#panel #panelActivities.panel-button > * {

  background-gradient-direction: none !important;
  border: none;
  color: transparent;
}

2.11. 更换 dock

  • 安装 gnome 扩展，https://extensions.gnome.org/extension/307/dash-to-dock/

解压之前下载的 OSX.for.Dash.to.DOCK 文件，打开文件 OSX.for.Dash.to.DOCK/Dock Settings/Gnome Version 3.26 - 3.28.md 文件，其中是一些 Command，现在打开终端，一次执行这些 Command: 下的命令吧，下面我把命令单独拿出来了，注意需要按顺序执行

gsettings set org.gnome.shell.extensions.dash-to-dock show-apps-at-top true

gsettings set org.gnome.shell.extensions.dash-to-dock custom-theme-running-dots false

gsettings set org.gnome.shell.extensions.dash-to-dock custom-theme-customize-running-dots false

gsettings set org.gnome.shell.extensions.dash-to-dock custom-theme-shrink false

gsettings set org.gnome.shell.extensions.dash-to-dock transparency-mode DEFAULT

gsettings set org.gnome.shell.extensions.dash-to-dock extend-height false

gsettings set org.gnome.shell.extensions.dash-to-dock dock-pozition BOTTOM

2.18. Plymouth 主题设置

Plymouth 主题也就是开机动画，可以在 https://www.gnome-look.org/browse/cat/108/ 这个类目下选择自己喜欢的开机动画主题进行安装。

  • cd /lib/;mkdir plymouth;cd plymouth/;sudo mkdir themes
  • 比如选择 https://www.gnome-look.org/p/1009320/ 这个主题，下载解压后，进入解压后的目录 170649-darwin，sudo cp darwin/ /lib/plymouth/themes/ -R
  • sudo update-alternatives --install /lib/plymouth/themes/default.plymouth default.plymouth /lib/plymouth/themes/darwin/darwin.plymouth 100，这条命令要根据你自己选择的主题的命令更改相应的位置
  • sudo update-alternatives --config default.plymouth

执行这条命令的时候要做相应的选择，如下图：

  • sudo update-initramfs -u
